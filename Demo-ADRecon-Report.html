▸ Critical S-DCSync DCSync rights granted to non-DC account Security 1 affected T1003.006
svc_backup holds "Replicating Directory Changes All" on the domain root. Any compromise of this account enables a full DCSync attack — dumping all NTLM hashes and Kerberos keys from the domain without touching a DC.
Affected Objects (1)
AccountPermissionObjectInherited
CORP\svc_backupReplicating Directory Changes AllDC=corp,DC=localFalse
Remediation Steps
  1. Open Active Directory Users and Computers → View → Advanced Features
  2. Right-click the domain root → Properties → Security → Advanced
  3. Remove "Replicating Directory Changes All" and "Replicating Directory Changes" from svc_backup
  4. Rotate the svc_backup credential immediately — assume it has been used for DCSync
  5. Review event logs for ID 4662 with GUID {1131f6aa-...} from non-DC sources

▸ Critical S-ESC1 ADCS ESC1 — certificate template allows arbitrary Subject Alternative Name Security 1 affected T1649
Template "UserTemplate-V2" on CA "CORP-CA" allows client-supplied Subject Alternative Names and grants enrollment to "Domain Users". Any authenticated user can request a certificate for Domain Admin and use it for PKINIT authentication — gaining a full TGT as DA without knowing their password.

▸ High P-Krbtgt krbtgt password not rotated in 847 days Accounts 1 affected T1558.001
The krbtgt account password has not been changed in 847 days (last changed: 2023-11-14). If an attacker obtained the krbtgt hash at any point since then, they can forge Golden Tickets valid for any user, any service, indefinitely — including bypassing MFA and Smart Card requirements. Golden Tickets forged with an old key remain valid until the key is changed TWICE.

▸ High S-GPPPassword GPP Passwords (MS14-025) — cPassword found in SYSVOL (2 files) Security 2 affected T1552.006
Group Policy Preferences XML files in SYSVOL contain cPassword values. Microsoft published the AES decryption key (KB2962486) — any domain user can decrypt these to recover plaintext credentials. This is exploited by Metasploit's post/windows/gather/credentials/gpp, PowerSploit's Get-GPPPassword, and CrackMapExec automatically.

▸ High P-UnconstrainedDelegation Unconstrained Kerberos delegation — 3 non-DC computer accounts Accounts 3 affected T1134.001
These machines have "Trust this computer for delegation to any service (Kerberos only)" enabled. When a privileged user (including Domain Admins) authenticates to any service on these machines, their TGT is cached in memory. A local administrator on any of these machines can extract these TGTs with Mimikatz and impersonate the victim. Combined with SpoolSample / PrinterBug, an attacker can coerce DC authentication to these machines.

  5. If SpoolSample/PrinterBug is a concern, disable the Print Spooler service on DCs
