▸ Critical AZ-SecurityDefaults Security Defaults disabled — no baseline MFA enforcement Identity T1078.004
Security Defaults are disabled and no equivalent Conditional Access policy enforcing MFA for all users was detected. Without Security Defaults or CA policies, users and admins can authenticate using only a password — making the tenant trivially compromised via password spray or credential stuffing. Microsoft reports Security Defaults block over 99.9% of identity attacks.
Remediation Steps
  1. If you have no existing CA policies: re-enable Security Defaults in Azure Portal → Azure AD → Properties → Manage Security Defaults
  2. If you have CA policies: create a policy requiring MFA for All Users / All Apps with no exclusions (or break-glass only)
  3. Ensure the MFA policy is in Enabled state — not Report-only
  4. Block legacy authentication protocols (Basic Auth) via a separate CA policy

▸ High AZ-LegacyAuth No Conditional Access policy blocking legacy authentication protocols ConditionalAccess T1078.004
No CA policy was found that blocks legacy authentication clients (Exchange ActiveSync, IMAP, POP3, SMTP AUTH, older Office clients). Legacy auth protocols do not support MFA — any MFA policy is bypassed if legacy auth is permitted. Microsoft reports 97% of password spray attacks use legacy authentication.
Remediation Steps
  1. Create a CA policy: Assignments → All users | Client apps → Exchange ActiveSync clients + Other clients | Grant → Block access
  2. Before enabling: check Sign-in logs for legacy auth usage (filter by Client App = "Exchange ActiveSync" etc.)
  3. Communicate to users that IMAP/POP3 access to Exchange Online will be disabled — migrate to Modern Auth clients
  4. Use Report-only mode for 2 weeks to assess impact before switching to Enabled

▸ Medium AZ-NoPIM Privileged Identity Management (PIM) not licensed — roles permanently assigned Identity T1078.004
No Azure AD P2 license was detected. Without PIM, all privileged role assignments are permanent — admins hold their roles 24/7 even when not performing administrative tasks. PIM enforces just-in-time activation with MFA, approval workflow, and time-limited sessions. Without it, a compromised admin account immediately has full privileges.
Remediation Steps
  1. License Azure AD P2 for all admin accounts (included in Microsoft 365 E5 or as add-on)
  2. Once licensed: migrate all permanent privileged role assignments to PIM eligible assignments
  3. Configure PIM activation requirements: MFA + justification + optional approval for GA role
  4. Set maximum activation duration: 8 hours for GA, 4 hours for other roles
  5. Enable PIM alerts for permanent assignments and unused eligible assignments
▸ Low AZ-SMSAuth SMS / voice call MFA still enabled in authentication methods policy AuthMethods T1621
SMS and voice call authentication methods are enabled for the tenant. SMS is vulnerable to SIM-swapping attacks — attackers bribe or social-engineer mobile carriers to transfer a victim's phone number, then receive their SMS OTP. CISA and NIST both recommend against SMS as an MFA factor for privileged accounts.
Remediation Steps
  1. Azure AD → Security → Authentication methods → SMS and Voice call → disable for all or admin-only group
  2. Migrate users from SMS to Microsoft Authenticator app (push notifications or passkey)
  3. For privileged accounts: require FIDO2 security keys or Windows Hello for Business
  4. Run MFA registration campaign: notify users to update their MFA methods before SMS is disabled
